# KoHs Crystal Tweaks 1.1.0-beta.4 — Minecraft 1.21.11

Fabric beta for Java 21, Yarn `1.21.11+build.5`, Fabric Loader `0.17.2`, and Fabric API `0.140.0+1.21.11`.

## Technical changes

- Fixed rapid place/attack cycles stalling because the crosshair could remain on a real crystal after that entity was removed client-side.
- Resolves the optimizer target from the outgoing attack packet's server-assigned entity ID, matching Marlow's reliable cleanup path.
- Clears the stale targeted entity and immediately retraces the crosshair after cleanup.
- Improves the upstream behavior by excluding KoHs local predictions during the retrace, allowing the next physical use to reach the valid block target.
- Retains the beta.3 pending-attack handoff for clicks that reach a local prediction before the real server entity exists.
- Preserves strict one-input/one-action behavior: no generated clicks, packet retries, guessed entity IDs, or extra attack packets.
- Adds an `Accept` / `Restore` confirmation before disabling every toggle.
- Shows every disable consequence first in English and then in Spanish while retaining compact hover descriptions.
- Includes Marlow's MIT attribution in both the repository and packaged JAR.

Rapid Attack Fix does not synthesize clicks, guess entity IDs, automate targets, or retry attacks. It preserves at most one vanilla-validated attack during the local-to-server crystal handoff.

Verified with `gradlew.bat clean build --no-daemon`. Minecraft was not launched.
