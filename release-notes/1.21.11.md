# KoHs Crystal Tweaks 1.1.0-beta.5 — Minecraft 1.21.11

Fabric beta for Java 21, Yarn `1.21.11+build.5`, Fabric Loader `0.17.2`, and Fabric API `0.140.0+1.21.11`.

## Technical changes

- Fixes crystal cycles that mix left mouse attack with Use Item bound to a keyboard key.
- Captures the arrival order of vanilla keyboard and mouse attack/use events, instead of allowing separate counters to reorder all attacks before all uses.
- Executes an ordered entry only after consuming one real count from the matching vanilla `KeyBinding.wasPressed()` queue.
- Immediately targets a locally predicted crystal after an accepted placement, enabling the next physical attack in the same client tick to enter Rapid Attack Fix.
- Retains the packet-ID cleanup, prediction-aware crosshair retrace, and bounded pending-attack handoff from beta.4.
- Does not remove the vanilla use cooldown, generate clicks, repeat held input, guess entity IDs, or add packets.
- Removes confirmations from Visual and Sound, plus `Static Crystal` and `Crystal Flotation` in Tweaks.
- Keeps bilingual English-first `Accept` / `Restore` warnings for `Local Crystal`, `Seamless Mode`, `Placement Fix`, and `Rapid Attack Fix`.
- Keeps all option explanations in responsive hover tooltips.

The ordered path is input-source neutral: direct keyboard bindings and mouse-button remaps use the same validation. It preserves one action for each actual vanilla press count and falls back to untouched vanilla processing if its bounded queue overflows.

Verified with `gradlew.bat clean build --no-daemon`. Minecraft was not launched.
