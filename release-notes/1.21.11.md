# KoHs Crystal Tweaks 2.0.0-beta.4 — Minecraft 1.21.11

Fabric release 2 beta for Java 21, Yarn `1.21.11+build.5`, Fabric Loader `0.17.2`, and Fabric API `0.140.0+1.21.11`.

## Placement Fix latency and ordering correction

- Preserves physical number-key and mouse-wheel hotbar changes together with attack/use presses.
- Replays each same-tick action with the item selected when that action physically occurred, instead of using Minecraft's final preprocessed slot for every use.
- Ignores keyboard repeat events and consumes one existing vanilla `KeyBinding.wasPressed()` count for every replayed attack or use.
- Creates a pending base only after vanilla accepts the obsidian placement.
- Never delays, retries, or later places a crystal input that arrived before the accepted obsidian interaction.
- Retargets only the recorded obsidian or its exact placement offset; adjacent-block guessing was removed.
- Fast-paths already-valid vanilla hits and rejects unrelated hits before pending-base entity collision checks, eliminating redundant scans.

## Default optimization state

Fresh installations and configurations missing these fields now use:

- `Local Crystal`: OFF
- `Seamless Mode`: OFF

Existing explicit values remain saved. With Local Crystal enabled, a predicted attack still waits only for the real server entity ID; KoHs does not guess IDs or generate an extra attack.

## Unchanged legitimacy guarantees

- No synthesized input, held-input automation, automatic retry, guessed entity ID, cooldown removal, remote target selection, or extra packet.
- Packet order follows the physical accepted sequence: obsidian first, then crystal.
- The server remains authoritative and may reject either interaction.

## Verification

- Clean Gradle build and all nine automated tests completed successfully.
- Minecraft was not launched.
